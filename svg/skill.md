skill.md

学前儿童识字 SVG 配图设计规范（可爱卡通风格）

⸻

1. 目标

为 3–6 岁学前儿童 生成高识别度、低干扰、强语义锚定的 可爱卡通风格 SVG 插画。

输出必须：
	•	单文件 SVG
	•	结构清晰
	•	可直接用于 Web / App
	•	统一卡通风格
	•	适配 1:1 容器
	•	第一眼可识别

⸻

2. 输出规格

2.1 画布
	•	viewBox="0 0 1024 1024"
	•	宽高比 1:1
	•	纯色或极浅柔和背景
	•	不使用位图
	•	不嵌入字体
	•	不依赖外部资源

⸻

2.2 固定图层结构（必须遵守）

<svg>
  <defs/>
  <g id="background"/>
  <g id="shadow"/>
  <g id="main-object"/>
  <g id="expression"/>
  <g id="accent"/>
</svg>

要求：
	•	id 名称固定
	•	不新增无意义 group
	•	不生成随机 id

⸻

3. 构图规则

3.1 主体规则
	•	单主体
	•	主体占画面 ≥ 65%
	•	主体居中
	•	禁止多主体叙事
	•	禁止复杂场景

⸻

3.2 儿童可爱风格原则
	•	圆润造型
	•	无锐利尖角
	•	线条端点 round
	•	适度夸张比例（头大、眼大）
	•	允许加入简单表情

⸻

4. 颜色规范（柔和童趣）

4.1 色彩数量
	•	主体 ≤ 4 色
	•	总色 ≤ 6 色
	•	禁止高饱和刺眼红绿对冲
	•	禁止复杂渐变
	•	禁止纹理

⸻

4.2 色彩风格
	•	柔和、明亮
	•	低对比但清晰
	•	可使用淡阴影（单色透明 8%–15%）
	•	禁止真实光影写实效果

⸻

5. 语义表达规则

5.1 名词类
	•	单个对象
	•	卡通化
	•	允许拟人表情（轻微）

示例策略：

猫 → 圆润小猫 + 微笑
山 → 圆顶小山 + 简单云朵（最多 1 个）

⸻

5.2 动词类
	•	表现“动作瞬间”
	•	允许少量速度线
	•	表情强化动作含义

示例：

跑 → 前倾姿态 + 微笑
跳 → 腾空 + 开心表情

⸻

5.3 抽象字

必须场景化 + 人物化表达。

禁止：
	•	纯符号
	•	抽象图形替代

示例：

爱 → 卡通人物拥抱
想 → 小朋友头顶思考气泡

⸻

6. 表情与情绪规范
	•	表情简洁
	•	眼睛：圆形或椭圆
	•	嘴巴：简单曲线
	•	禁止复杂五官细节
	•	禁止写实鼻子结构

⸻

7. SVG 技术要求
	•	使用基础图形（circle / rect / path）
	•	stroke-width 统一
	•	使用 round linecap
	•	不写死 px 单位
	•	不写 transform matrix
	•	保持结构可动画

⸻

8. 禁止事项
	•	不写文字
	•	不写拼音
	•	不添加说明
	•	不添加多余背景元素
	•	不生成复杂滤镜
	•	不使用 filter / blur
	•	不做多对象故事场景

⸻

9. AI 生成 Prompt 模板

为学前儿童设计一个可爱卡通风格的识字 SVG 插画，
汉字为「{字}」。

要求：
- 1024x1024 画布
- 单主体
- 主体占比 65%以上
- 圆润造型
- 可爱卡通风格
- 柔和配色
- 不超过 6 种颜色
- 不要文字
- 不要拼音
- 不要复杂背景
- 输出纯 SVG 代码


⸻

10. 自检清单

生成后必须检查：
	•	是否一眼能认出对应字义
	•	是否足够可爱
	•	是否圆润无尖锐
	•	是否颜色过多
	•	是否背景干扰主体
	•	是否缩小到 200px 仍清晰
	•	是否符合学前儿童审美

⸻

11. 文件命名规范

hanzi_{pinyin}.svg

示例：

hanzi_mao.svg
hanzi_shan.svg
hanzi_ai.svg


⸻

12. 版本控制
	•	v1：基础可爱风格
	•	v2：优化比例与表情
	•	v3：增加轻微动效适配结构
